## ЁЯЪА Quick Start

```bash
# Clone the repository
git clone https://github.com/LegendarySumit/git-first-project.git
cd git-first-project

# Install GitHub CLI if not already installed
# Windows users can use Winget or installer from https://cli.github.com/

# Authenticate GitHub CLI
gh auth login

# Create a new branch and switch to it
git switch -c feature/branching-practice

# Add a change and commit
echo "# Git practice" >> README.md
git add README.md
git commit -m "chore: add branching practice note"

# Push the branch to GitHub
git push -u origin feature/branching-practice

# Merge the branch back into main
git switch main
git pull origin main
git merge feature/branching-practice
git push origin main
```

## тЪЩя╕П Configuration

### GitHub CLI
- Run `gh auth login` to authenticate your GitHub account.
- Use `gh repo create git-first-project --public --source=. --remote=origin --push` to publish this repository from terminal.

### Recommended Git config
```bash
git config --global user.name "LegendarySumit"
git config --global user.email "your-email@example.com"
git config --global init.defaultBranch main
```

### Branch workflow
- `git switch -c feature/<name>` тАФ create and switch to a new feature branch
- `git switch main` тАФ switch back to the main branch
- `git merge feature/<name>` тАФ merge the branch into main

## ЁЯРЫ Troubleshooting

- **`git: unknown command 'switch'`** тАФ update Git to a version that supports `git switch`.
- **`gh: command not found`** тАФ install GitHub CLI from https://cli.github.com/.
- **Authentication fails** тАФ run `gh auth login` again, then verify with `gh auth status`.
- **Merge conflicts** тАФ resolve conflicts in files, then `git add <file>` and `git commit`.
